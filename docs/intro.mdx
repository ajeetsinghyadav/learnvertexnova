## What is VertexNova?

**VertexNova Engine** is a modern, multi-platform rendering and visualization engine designed for **learning, clarity, and long-term maintainability**. Unlike commercial game engines or high-level graphics libraries, VertexNova provides:

- **Transparent architecture** â€” See exactly how rendering works, from API calls to frame presentation
- **Multi-backend support** â€” Learn how to abstract across Vulkan, Metal, OpenGL, and WebGL with a unified API
- **Clean C++ design** â€” Modern C++17+ with strict ownership, predictable lifetimes, and clear abstractions
- **Educational focus** â€” Built to teach, not just to ship products

### Why not Unity, Unreal, or other engines?

**VertexNova is fundamentally different:**

| Aspect              | Unity/Unreal              | VertexNova                    |
|---------------------|---------------------------|-------------------------------|
| **Purpose**         | Ship games quickly        | Learn engine architecture     |
| **Abstraction Level** | High-level, black-box   | Low-level, transparent        |
| **Code Visibility** | Mostly hidden             | Fully open and readable       |
| **Learning Value**  | Learn game design         | Learn graphics programming    |
| **Backend Control** | Limited                   | Full control over all backends |
| **Complexity**      | Millions of lines         | Focused, manageable codebase  |

**Why not other graphics libraries?**

- **GLFW/SDL**: Great for windowing, but you still need to write all rendering code yourself

- **bgfx/LLGL**: Excellent abstractions, but less educational â€” the magic is hidden

- **Raw Vulkan/Metal/OpenGL**: Too low-level, too much boilerplate, hard to learn

**VertexNova strikes the perfect balance**: enough abstraction to be practical, enough transparency to be educational, and enough structure to build real projects.

## Architecture overview

<figure>
  <img src="/img/diagrams/vertexnova_layers.png" alt="VertexNova architecture layers" />
  <figcaption>VertexNova layers: platform (xwin) + unified graphics API (xgl) + backend implementations.</figcaption>
</figure>

VertexNova is organized into clean, focused components:

- **xwin** â€” Platform and window/input utilities abstracting OS-specific functionality
- **xgl** â€” Multi-backend rendering layer providing a unified API across OpenGL, Metal, Vulkan, and WebGL
- **xviz** â€” Visualization helpers and utilities for common graphics tasks

This architecture is **intentionally simple**. Each component has a clear responsibility, making the codebase easy to understand, modify, and extend.

## Design philosophy

VertexNova follows a clear set of principles that guide every design decision:

- **Modern C++ with clarity over cleverness** â€” Readable code is more valuable than clever code
- **Thin abstractions and clear layering** â€” Enough abstraction to be practical, not so much that it's magic
- **Multi-backend design via unified API** â€” Learn once, run everywhere
- **Build to learn first; scale responsibly later** â€” Education and understanding come before optimization
