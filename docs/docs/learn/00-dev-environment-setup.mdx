# 00: Dev Environment Setup

Before you build anything, you need a working development environment. This page tells you what to install and points you to the canonical **[vertexnova-devsetup](https://github.com/ajeetsinghyadav/vertexnova-devsetup)** repository for step-by-step instructions per platform.

![Dev environment — tools per platform: Linux, Windows, macOS](/img/learn/diagrams/00_dev_stack.png)

## What you need

| Category | Linux (Ubuntu 22.04 / 24.04) | Windows | macOS |
|----------|------------------------------|---------|--------|
| **Compiler** | GCC ≥ 11 or Clang ≥ 13 | MSVC (Visual Studio) | Xcode / Apple Clang |
| **Build** | CMake ≥ 3.20, Git | CMake, Git | CMake, Xcode CLI tools, Git |
| **Graphics (Vulkan)** | Vulkan SDK, Mesa / GPU drivers | Vulkan SDK, GPU drivers | MoltenVK (Vulkan on Metal) |
| **Graphics (Metal)** | — | — | Built into Xcode |
| **Graphics (OpenGL)** | Mesa / GPU drivers | GPU drivers | Built-in |
| **IDE** | VS Code, Qt Creator | Visual Studio, VS Code, Qt Creator | Xcode, VS Code, Qt Creator |
| **Tools** | Python ≥ 3.8 | Python ≥ 3.8 | Python ≥ 3.8 |
| **Optional** | Doxygen, clang-format, clang-tidy, MeshLab | Doxygen, clang-format, MeshLab | Doxygen, clang-format, MeshLab |

Details — versions, install commands, driver links — are in the devsetup repository for each platform.

## What each guide covers

The devsetup guides walk you from a clean machine to a working build. Each one covers:

| Topic | What it sets up |
|-------|----------------|
| Compiler | GCC/Clang (Linux, macOS), MSVC (Windows), minimum version checks |
| Vulkan SDK | Install, `PATH`, validation layers, `VK_LAYER_PATH` |
| MoltenVK | Vulkan-on-Metal translation layer for macOS |
| IDEs | VS Code + extensions, Visual Studio (Windows), Qt Creator |
| Python | Scripts and tooling (≥ 3.8) |
| Drivers | GPU drivers where needed — Linux and Windows especially |
| Optional | MeshLab, Doxygen, clang-format, clang-tidy |

Content in the repository may be updated over time; if something on this page is out of date, the devsetup repo and its linked guides are the source of truth.
