# 01: Hello VertexNova

Every VertexNova project starts from **[vnetemplate](https://github.com/vertexnova/vnetemplate)** — an official project template that already includes the build system, internal dependencies (vnelogging, vnecommon), tests, and CI configuration. Clone it, build, and run. In the next chapter you will use the logging that is already inside this project.

## Quick Start

```bash
# 1. Clone
git clone https://github.com/vertexnova/vnetemplate.git
cd vnetemplate

# 2. Initialize dependencies
git submodule update --init --recursive

# 3. Build (choose one)
./scripts/build_macos.sh      # macOS
./scripts/build_linux.sh      # Linux
./scripts/build_windows.ps1   # Windows (PowerShell)

# 3b. Or build with CMake directly
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build

# 4. Run
./build/bin/<example_name>
```

| Command | What it does |
|---------|-------------|
| `git submodule update --init --recursive` | Pulls all dependencies — vnelogging, vnecommon, googletest, and their transitive submodules. |
| `cmake -B build` | Configures the project and writes build files into `build/`. |
| `-DCMAKE_BUILD_TYPE=Release` | Builds optimized. Use `Debug` for a debuggable binary with symbols. |
| `cmake --build build` | Compiles the project. |

## Template layout

| Path | Purpose |
|------|---------|
| `cmake/vnecmake/` | Shared CMake helpers (compiler flags, sanitizers, install rules). |
| `deps/internal/` | Internal VertexNova dependencies: vnelogging, vnecommon. |
| `deps/external/` | External dependencies: googletest. |
| `include/` | Public headers for your library or app. |
| `src/` | Implementation files. |
| `tests/` | Unit tests using Google Test. |
| `examples/` | Runnable example programs. |
| `scripts/` | One-command build scripts for macOS, Linux, and Windows. |
| `.clang-format` | Code style rules — enforced automatically in CI. |
| `.clang-tidy` | Static analysis checks — enforced in CI. |
| `.github/workflows/` | CI pipeline: format check, build, tests on all platforms. |

## Starting a new project

vnetemplate is the same structure used to create every VertexNova library. When you start something new: clone it, rename the project and targets in `CMakeLists.txt`, and write your code in `include/` and `src/`. The build system, dependencies, tests, and CI are all already wired up.

## Build from scratch (optional)

If you want to understand every line before using the template, a minimal VertexNova project needs three things:

**`src/main.cpp`**

```cpp
#include <iostream>

int main() {
    std::cout << "Hello, VertexNova!" << std::endl;
    return 0;
}
```

**`CMakeLists.txt`**

```cmake
cmake_minimum_required(VERSION 3.16)
project(HelloVertexNova LANGUAGES CXX)
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_RUNTIME_OUTPUT_DIRECTORY ${CMAKE_BINARY_DIR}/bin)
add_executable(HelloVertexNova src/main.cpp)
```

**Build and run:**

```bash
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
./build/bin/HelloVertexNova
# Hello, VertexNova!
```

Once this works, [02: Adding Logging](../02-adding-logging) shows how to add vnelogging to this project as a submodule or via FetchContent.
