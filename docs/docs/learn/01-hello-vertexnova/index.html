01: Hello VertexNova

Now that your environment is set up (from 00: Dev Environment Setup), you'll get a runnable VertexNova project by cloning the official project template. You don't need to know CMake — we'll use a few commands and explain them in one line.

This is the same template used to create new VertexNova libraries (e.g. vnelogging). It already includes vnelogging in deps/internal/, so in the next chapter we'll use the logging that's already there.

What We're Building​

You will:

  1. Clone vnetemplate
  2. Initialize its submodules (vnelogging, vnecommon, etc.)
  3. Build using the template's scripts or CMake
  4. Run the template's example binary

Outcome: you have a real VertexNova project that builds and runs.

Step 1: Clone the Template​

git clone https://github.com/vertexnova/vnetemplate.git
cd vnetemplate

Step 2: Initialize Submodules​

The template uses Git submodules for dependencies (vnelogging, vnecommon, Google Test). Initialize them:

git submodule update --init --recursive

Step 3: Build​

Use the template's build scripts (recommended) or CMake directly.

Option A — Scripts (macOS / Linux / Windows):

# macOS
./scripts/build_macos.sh

# Linux
./scripts/build_linux.sh

# Windows (PowerShell)
./scripts/build_windows.ps1

Option B — CMake directly:

cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
  • cmake -B build configures the project (generates build files).
  • cmake --build build compiles the code.

Step 4: Run the Example​

After a successful build, run the example binary. Typical locations:

# If the template provides an example executable (path may vary; check build/bin or build/)
./build/bin/<example_name>

If the template has a single main example, it might be under build/bin/ — list the directory to see the exact name, then run it. You should see output from your first VertexNova program.

What's in the Template?​

PathPurpose
cmake/vnecmake/Shared CMake helpers (vnecmake)
deps/internal/Optional internal deps: vnelogging, vnecommon
deps/external/External deps (e.g. googletest)
include/, src/Your library or app code
tests/, examples/Tests and example apps
scripts/Build scripts for macOS, Linux, Windows
.clang-format, .clang-tidyCode style and static analysis

The template is ready for CI (format, tidy, build/test). When you start a new VertexNova library, you clone vnetemplate, rename it to your project, and change the code — no hand-written CMake from zero.

What's Next?​

In 02: Adding Logging we'll use the logging that's already in this project: where it lives (deps/internal/vnelogging), how to call it (VNE_LOG_*, LoggerConfig), and how to run the hello-logging example.

Checkpoint: You have a built template project and a runnable binary. Same structure and tools that real VertexNova libraries use.

Optional: Build from Scratch​

If you prefer to see every line of a minimal project instead of using the template, follow these steps. Each one is self-contained so you can copy and adapt as needed.

Step 1 — Create the project folder and main file

Create a project directory (e.g. HelloVertexNova) and a source file at src/main.cpp.

Create the folder:

mkdir -p HelloVertexNova/src
cd HelloVertexNova

Add src/main.cpp with a minimal C++ program:

#include <iostream>

int main() {
    std::cout << "Hello, VertexNova!" << std::endl;
    return 0;
}
Step 2 — Add a minimal CMakeLists.txt

In the project root (same folder as src/), create CMakeLists.txt with at least:

  • cmake_minimum_required(VERSION 3.16) — required CMake version
  • project(HelloVertexNova LANGUAGES CXX) — project name and C++ only
  • set(CMAKE_CXX_STANDARD 17) — use C++17
  • add_executable(HelloVertexNova src/main.cpp) — build one executable from main.cpp

Example CMakeLists.txt:

cmake_minimum_required(VERSION 3.16)
project(HelloVertexNova LANGUAGES CXX)
set(CMAKE_CXX_STANDARD 17)
add_executable(HelloVertexNova src/main.cpp)

Optional: to put the binary in build/bin/, add
set(CMAKE_RUNTIME_OUTPUT_DIRECTORY ${CMAKE_BINARY_DIR}/bin) before add_executable.

Step 3 — Configure and build

From the project root, configure and build in a separate build directory:

cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
  • cmake -B build — configure the project and generate build files into build/.
  • -DCMAKE_BUILD_TYPE=Release — build optimized (use Debug for debugging).
  • cmake --build build — compile the executable.
Step 4 — Run the executable

Run the program. The executable name matches the first argument to add_executable (here, HelloVertexNova).

If you did not set CMAKE_RUNTIME_OUTPUT_DIRECTORY:

./build/HelloVertexNova

If you put the binary in build/bin/:

./build/bin/HelloVertexNova

You should see: Hello, VertexNova!

If you built from scratch, 02: Adding Logging has an optional section on adding vnelogging to that project (submodule or FetchContent).
