00: Dev Environment Setup

Before you build a single demo or write a line of engine code, you need a working development environment. This page summarizes what you need and points you to platform-specific setup guides. You don't need to know CMake — we'll use a few commands and explain them when we need them.

The canonical setup repository is vertexnova-devsetup — a guide to setting up your laptop for VertexNova engine development. Use it for step-by-step instructions per OS.

What Each Guide Covers​

The devsetup repo is meant to get you from a clean machine to a working build. Typical topics:

  • Compilers — GCC/Clang (Linux, macOS), MSVC (Windows), minimum versions
  • Vulkan SDK — Install and environment (PATH, layers)
  • MoltenVK — Using Vulkan on macOS (via MoltenVK)
  • IDEs — Visual Studio (Windows), VS Code, Qt Creator, Xcode (macOS)
  • Python — For scripts and tooling (e.g. ≥ 3.8)
  • MeshLab — Optional, for inspecting/processing meshes
  • Doxygen — Optional, for generating API docs
  • clang-format / clang-tidy — Optional, for code style and static analysis
  • Drivers — GPU drivers where needed (especially Linux and Windows)

Content in the repo may be updated over time; if something here is out of date, the vertexnova-devsetup README and the linked docs are the source of truth.
