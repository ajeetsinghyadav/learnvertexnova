# 04: Scene - Cameras and Lights

**vnescene** is three focused pieces: a **camera** that defines what is in view, **lights** that describe how geometry is illuminated, and a **SceneState** container that holds the active camera and a list of lights. It does not own geometry or GPU resources — its output is GPU-packed structs (`CameraGpu`, `LightGpu`) that you upload to a uniform buffer.

The library providing this is [vnescene](https://github.com/vertexnova/vnescene). The [vnetestbed](https://github.com/vertexnova/vnetestbed) `02_test_scene` sample runs the full pipeline interactively.

## What vnescene gives you

| Component | What it is |
|-----------|-----------|
| `PerspectiveCamera` | FOV-based projection. position, target, up, near/far, aspect. |
| `OrthographicCamera` | Parallel projection. left/right/bottom/top or half-extent + near/far. |
| `CameraGpu` | Packed struct (view, projection, view-projection, position) for GPU upload. |
| `camera_utils` | `fitToAabb`, `screenToWorldRay`, `project`, `unproject`, `buildFrustum`. |
| `AmbientLight` | Non-directional fill; color + intensity. |
| `DirectionalLight` | Parallel rays (sun); direction, color, intensity. |
| `PointLight` | Omni; position, color, intensity, range. |
| `SpotLight` | Cone; position, direction, color, intensity, range, inner/outer angle. |
| `LightGpu` | Packed struct for all light types. |
| `SceneState` | Holds one active camera and a list of lights. |
| `Environment` | Exposure, gamma, clear color, optional HDRI id (data only). |

Full reference: **[vnescene repository](https://github.com/vertexnova/vnescene)**.

## How the camera works

Before any vertex reaches the screen, three matrices transform it from model space to clip space. vnescene cameras compute and cache all three.

![World → camera → clip → screen — camera spaces](/img/learn/diagrams/04_camera_spaces.png)

The **view matrix** moves the world so the camera is at the origin, looking down -Z. The **projection matrix** maps the view frustum into a unit cube (NDC). For OpenGL the Z range is [-1, 1]; for Vulkan it is [0, 1]. Call `camera.setGraphicsApi(vne::math::GraphicsApi::eVulkan)` to get the correct projection — the camera handles the difference internally.


The camera does not own the viewport transform — that lives in the graphics API (OpenGL `glViewport`, Vulkan `VkViewport`).

### View matrix construction

<!-- Add static/img/learn/diagrams/04_view_matrix.png (export from static/img/learn/diagrams/04_view_matrix.excalidraw) -->
![View matrix — lookAt basis vectors and camera frame](/img/learn/diagrams/04_view_matrix.png)

The view matrix is built from three vectors derived from `position`, `target`, and `up`:

$$
\mathbf{f} = \text{normalize}(\text{target} - \text{eye})
\qquad
\mathbf{r} = \text{normalize}(\mathbf{f} \times \mathbf{up})
\qquad
\mathbf{u} = \mathbf{r} \times \mathbf{f}
$$

These form an orthonormal basis. The view matrix rotates the world into camera space and then translates by the negated eye position:

$$
V =
\begin{bmatrix}
r_x & r_y & r_z & -\mathbf{r}\cdot\text{eye} \\
u_x & u_y & u_z & -\mathbf{u}\cdot\text{eye} \\
-f_x & -f_y & -f_z & \mathbf{f}\cdot\text{eye} \\
0 & 0 & 0 & 1
\end{bmatrix}
$$

The camera looks down $-Z$ in camera space. The inverse $V^{-1} = R \cdot T(\text{eye})$ transforms from camera space back to world space — used internally by `screenToWorldRay`.

## Perspective vs orthographic

![Perspective vs orthographic projection](/img/learn/diagrams/04_camera_projection.png)

A **perspective camera** models a pinhole — parallel lines converge. Depth is non-linear in NDC. Use it for 3D scenes where you want depth cues and foreshortening.

An **orthographic camera** keeps parallel lines parallel. Scale does not change with depth. Use it for 2D, UI, technical/CAD views, and shadow maps.

### Conventions used here

Before the matrices: a few terms that appear throughout.

**Handedness** — OpenGL, Metal, and vnescene use a **right-handed** coordinate system: $X$ right, $Y$ up, $Z$ toward the viewer. Camera space has the camera looking down $-Z$.

**NDC depth range** — different APIs map the view frustum to different depth intervals:

| API | NDC Z range | Y axis in NDC |
|-----|------------|---------------|
| OpenGL | $[-1, 1]$ | up |
| Vulkan | $[0, 1]$ | **down** (flipped) |
| Metal | $[0, 1]$ | up |

Call `camera->setGraphicsApi(vne::math::GraphicsApi::eVulkan)` (or `eMetal`) — the library rebuilds the projection to match the chosen convention.

**Matrix layout** — vnescene stores matrices **column-major**, matching GLSL/HLSL `layout(column_major)` and Metal's default. When you read the entries below, column $j$ of the matrix is stored as the $j$-th group of four floats in memory. Row vectors (e.g. DirectX HLSL `row_major`) would be the transpose.

**Transform order** — to combine scale, rotation, and translation into a model matrix use $M = T \times R \times S$ (right to left: scale first, rotate, then translate). Applying scale after rotation produces shear.

---

### Perspective projection matrix

Let $e = 1/\tan(\text{fov}/2)$ and $a$ = aspect ratio ($a = w/h$). The frustum is defined by $n$ (near), $f$ (far), and the FOV. OpenGL right-handed, $Z \in [-1, 1]$:

$$
P_{\text{persp}} =
\begin{bmatrix}
e/a & 0 & 0 & 0 \\
0 & e & 0 & 0 \\
0 & 0 & \displaystyle-\frac{f+n}{f-n} & \displaystyle-\frac{2fn}{f-n} \\
0 & 0 & -1 & 0
\end{bmatrix}
$$

The $-1$ in row 3 col 2 (0-indexed) produces the **perspective divide**: after the matrix multiply, $w_\text{clip} = -z_\text{view}$. Hardware divides $xyz$ by $w$ to get NDC, making depth non-linear (more precision near $n$, less near $f$).

**Frustum form** — if you have explicit frustum bounds $[l, r] \times [b, t]$ at the near plane instead of FOV:

$$
P_{\text{frustum}} =
\begin{bmatrix}
\displaystyle\frac{2n}{r-l} & 0 & \displaystyle\frac{r+l}{r-l} & 0 \\[6pt]
0 & \displaystyle\frac{2n}{t-b} & \displaystyle\frac{t+b}{t-b} & 0 \\[6pt]
0 & 0 & \displaystyle-\frac{f+n}{f-n} & \displaystyle-\frac{2fn}{f-n} \\[6pt]
0 & 0 & -1 & 0
\end{bmatrix}
$$

For a symmetric frustum ($l = -r$, $b = -t$) this collapses to the FOV form above.

**Vulkan / Metal variant** — Vulkan flips $Y$ (row 1 negated) and shifts depth to $[0, 1]$ (row 2 changes). Metal shifts depth to $[0, 1]$ but keeps $Y$ up. `setGraphicsApi` handles both.

---

### Orthographic projection matrix

**Explicit bounds form** — for a box $[l, r] \times [b, t] \times [n, f]$. OpenGL, $Z \in [-1, 1]$:

$$
P_{\text{ortho}} =
\begin{bmatrix}
\displaystyle\frac{2}{r-l} & 0 & 0 & \displaystyle-\frac{r+l}{r-l} \\[6pt]
0 & \displaystyle\frac{2}{t-b} & 0 & \displaystyle-\frac{t+b}{t-b} \\[6pt]
0 & 0 & \displaystyle-\frac{2}{f-n} & \displaystyle-\frac{f+n}{f-n} \\[6pt]
0 & 0 & 0 & 1
\end{bmatrix}
$$

No perspective divide — $w_\text{clip} = 1$ always, so depth is linear and objects do not change size with distance.

**Half-extent form** — vnescene's second `OrthographicCamera` constructor takes a half-extent $h$ and viewport size. It computes symmetric bounds from the aspect ratio $a = w/h_\text{px}$:

$$
l = -a \cdot h,\quad r = +a \cdot h,\quad b = -h,\quad t = +h
$$

then substitutes into the explicit-bounds matrix above. Useful for 2D cameras where you want to express "show $h$ world units above and below center."

**Vulkan / Metal variant** — the $Z$ row becomes $\tfrac{-1}{f-n}$ and the translation shifts to $\tfrac{-n}{f-n}$ for $[0, 1]$ depth. `setGraphicsApi` handles this.

## Quick Start

```cpp
#include <vertexnova/scene/scene.h>

// --- Camera ---
auto camera = std::make_shared<vne::scene::PerspectiveCamera>(
    60.0f,          // FOV (degrees)
    1280.0f,        // viewport width  →  aspect = width/height
    720.0f,         // viewport height
    0.1f,           // near plane
    1000.0f         // far plane
);
camera->setPosition({0.0f, 3.0f, 8.0f});
camera->setTarget({0.0f, 0.0f, 0.0f});

// --- Lights ---
auto ambient = std::make_shared<vne::scene::AmbientLight>(
    vne::math::Vec3f{0.08f, 0.08f, 0.1f},  // color
    1.0f                                    // intensity
);
auto sun = std::make_shared<vne::scene::DirectionalLight>(
    vne::math::Vec3f{-0.5f, -1.0f, -0.3f},  // direction (world space, not normalized required)
    vne::math::Vec3f{1.0f, 0.97f, 0.9f},    // color
    1.0f                                     // intensity
);

// --- Scene state ---
vne::scene::SceneState scene;
scene.setActiveCamera(camera);
scene.addLight(ambient);
scene.addLight(sun);

// --- GPU pack ---
if (scene.hasActiveCamera()) {
    vne::scene::CameraGpu cam_gpu = scene.getActiveCamera()->toGpu();
    // upload cam_gpu to a UBO / uniform block
}
for (auto& light : scene.getLights()) {
    vne::scene::LightGpu lg = light->toGpu();
    // append lg to a light array buffer
}
```

| Call | What it does |
|------|-------------|
| `PerspectiveCamera(fov, w, h, near, far)` | Computes aspect = w/h; matrices are lazy-computed on first get. |
| `camera->setPosition / setTarget` | Mark matrices dirty; recomputed on next `getViewMatrix()`. |
| `camera->toGpu()` | Returns `CameraGpu` — view, projection, viewProj, position. Upload as a UBO. |
| `scene.addLight(ptr)` | Appends; respects `setMaxLights` cap if set. |
| `light->toGpu()` | Returns `LightGpu` — type tag + all fields — same struct for all light types. |

## Camera types and construction

### PerspectiveCamera

```cpp
// From FOV + explicit aspect
vne::scene::PerspectiveCamera cam(60.0f, 16.0f/9.0f, 0.1f, 1000.0f);

// From FOV + viewport size (aspect computed)
vne::scene::PerspectiveCamera cam(60.0f, 1280.0f, 720.0f, 0.1f, 1000.0f);

// Common mutations
cam.setFieldOfView(45.0f);
cam.setNearPlane(0.01f);
cam.setFarPlane(500.0f);
cam.resize(width, height);           // call on window resize
cam.lookAt({0.0f, 0.0f, 0.0f});     // point at target, keep current up

// First-person / orbit helpers
cam.moveForward(delta * speed);
cam.moveRight(delta * speed);
cam.rotateAroundTarget(yaw_deg, pitch_deg);  // orbit

// Navigation helpers
cam.getForward();   // normalized forward vector
cam.getRight();     // normalized right vector
```

### OrthographicCamera

```cpp
// From explicit frustum bounds
vne::scene::OrthographicCamera cam(-8.0f, 8.0f, -4.5f, 4.5f, -100.0f, 100.0f);

// From half-extent (symmetric, aspect from width/height)
vne::scene::OrthographicCamera cam(6.0f, 1280.0f, 720.0f, -100.0f, 100.0f);
// → left=-aspect*half, right=+aspect*half, bottom=-half, top=+half
```

### Via CameraFactory

```cpp
vne::scene::CameraParameters params;
params.type         = vne::scene::CameraType::ePerspective;
params.fov          = 60.0f;
params.aspect_ratio = 16.0f / 9.0f;
params.near_plane   = 0.1f;
params.far_plane    = 1000.0f;

auto camera = vne::scene::CameraFactory::create(params);
```

### Viewport resize

Call `camera->resize(width, height)` or `camera->setViewport(width, height)` when the window size changes. This recomputes aspect and marks projection dirty.

### Graphics API

```cpp
// Default is OpenGL. Switch for Vulkan (flips Y in projection, shifts Z range to [0,1])
camera->setGraphicsApi(vne::math::GraphicsApi::eVulkan);
```

## Light types

All light types share the same base diffuse term — $\mathbf{N}$ is the surface normal, $\mathbf{L}$ points toward the light:

$$
L_{\text{diffuse}} = \max(\mathbf{N} \cdot \mathbf{L},\; 0) \times \text{color} \times \text{intensity}
$$

### AmbientLight — fill illumination

No position or direction. Applied as a constant base term so unlit surfaces are never pure black:

$$
L_{\text{ambient}} = \text{color} \times \text{intensity}
$$

```cpp
auto ambient = std::make_shared<vne::scene::AmbientLight>(
    vne::math::Vec3f{0.08f, 0.08f, 0.1f},  // color
    1.0f                                    // intensity
);
```

### DirectionalLight — parallel rays (sun, sky)

![Directional light — parallel rays, no position](/img/learn/diagrams/04_light_directional.png)

Direction points toward the light source (opposite of travel). No position, no attenuation — illumination is equal everywhere:

$$
\mathbf{L} = \text{normalize}(-\text{direction})
\qquad
L_{\text{dir}} = \max(\mathbf{N} \cdot \mathbf{L},\; 0) \times \text{color} \times \text{intensity}
$$

```cpp
auto sun = std::make_shared<vne::scene::DirectionalLight>(
    vne::math::Vec3f{-0.5f, -1.0f, -0.3f},  // direction
    vne::math::Vec3f{1.0f, 0.97f, 0.9f},    // warm white
    1.0f
);
```

### PointLight — omni light with range

![Point light — omni from a position with range](/img/learn/diagrams/04_light_point.png)

Illuminates in all directions from a position. Intensity falls off with distance $d = \lVert \text{lightPos} - \text{fragPos} \rVert$ to zero at `range`:

$$
a(d) = \left(1 - \text{clamp}\!\left(\frac{d}{\text{range}},\; 0,\; 1\right)^2\right)^2
\qquad
L_{\text{point}} = \max(\mathbf{N} \cdot \mathbf{L},\; 0) \times \text{color} \times \text{intensity} \times a(d)
$$

```cpp
auto lamp = std::make_shared<vne::scene::PointLight>(
    vne::math::Vec3f{0.0f, 3.0f, 0.0f},  // position
    vne::math::Vec3f{1.0f, 0.8f, 0.4f},  // warm color
    2.0f,                                 // intensity
    8.0f                                  // range (world units)
);
lamp->setPosition({2.0f, 2.0f, 0.0f});   // move it
lamp->setRange(12.0f);
```

### SpotLight — cone light

![Spot light — cone with inner/outer angles](/img/learn/diagrams/04_light_spot.png)

Position + direction + two cone angles. The library stores the angles as cosines: $c_i = \cos(\theta_\text{inner})$, $c_o = \cos(\theta_\text{outer})$. Inside the inner cone: full intensity. Between inner and outer: smooth falloff. Beyond outer: no contribution. **Outer angle must be greater than inner angle.**

$$
s = \text{smoothstep}\!\left(0,\; 1,\; \frac{\mathbf{L}\cdot\text{normalize}(-\text{dir}) - c_o}{c_i - c_o}\right)
\qquad
L_{\text{spot}} = \max(\mathbf{N} \cdot \mathbf{L},\; 0) \times \text{color} \times \text{intensity} \times a(d) \times s
$$

```cpp
auto spot = std::make_shared<vne::scene::SpotLight>(
    vne::math::Vec3f{0.0f, 4.0f, 4.0f},   // position
    vne::math::Vec3f{0.0f, -0.7f, -0.7f}, // direction
    vne::math::Vec3f{1.0f, 0.9f, 0.7f},   // color
    2.0f,                                  // intensity
    10.0f,                                 // range
    15.0f,                                 // inner angle (degrees)
    30.0f                                  // outer angle (degrees)
);
```

## Light summary

![Light types overview — ambient, directional, point, spot](/img/learn/diagrams/04_light_types.png)

| Type | Position | Direction | Range | Cone | Use |
|------|----------|-----------|-------|------|-----|
| Ambient | — | — | — | — | Base fill, prevents total black |
| Directional | — | yes | — | — | Sun, sky, dominant light source |
| Point | yes | — | yes | — | Lamp, torch, explosion, any local omni |
| Spot | yes | yes | yes | yes | Flashlight, stage light, car headlight |

`LightGpu` is a 64-byte std140-compatible struct. All four types share the same layout — the type tag in `misc.x` tells the shader which fields are active:

| Field | Content |
|-------|---------|
| `color_intensity` | $(r, g, b,\; \text{intensity})$ |
| `position_range` | $(\text{pos.xyz},\; \text{range})$ |
| `direction_inner_cos` | $(\text{dir.xyz},\; c_i)$ |
| `misc` | $(\text{type},\; \text{enabled},\; c_o,\; 0)$ — type: $0$=ambient $1$=dir $2$=pt $3$=spot |

## SceneState

```cpp
vne::scene::SceneState scene;

// Camera
scene.setActiveCamera(camera);
auto* cam = scene.getActiveCamera().get();  // ICamera*
bool has_cam = scene.hasActiveCamera();

// Lights
scene.addLight(ambient);
scene.addLight(sun);
scene.removeLight(sun);     // by shared_ptr
scene.clearLights();
size_t n = scene.getLightCount();
const auto& lights = scene.getLights();  // std::vector<shared_ptr<ILight>>

// Optional cap: keep at most 8 lights
scene.setMaxLights(8);
```

## Camera utilities

```cpp
#include <vertexnova/scene/camera/camera_utils.h>

// World-space ray from a screen pixel (for picking)
auto ray = vne::scene::screenToWorldRay(*camera, mouse_x, mouse_y, 1280.0f, 720.0f);
// ray.origin = camera position, ray.direction = world direction through pixel

// Fit camera to bounding box (model viewer / focus-on-object)
vne::math::Aabb box = compute_scene_aabb();
vne::scene::fitToAabb(*persp_camera, box, 1.1f);  // 1.1 = 10% padding

// Project world → screen
auto screen_pt = vne::scene::project(*camera, world_pos, 1280.0f, 720.0f);

// Unproject screen → world
auto world_pt = vne::scene::unproject(*camera, {mouse_x, mouse_y, depth}, 1280.0f, 720.0f);

// Frustum for culling
auto frustum = vne::scene::buildFrustum(*camera);
```

## Environment

```cpp
vne::scene::Environment env;
env.exposure = 1.0f;
env.gamma    = 2.2f;
env.setClearColor({0.1f, 0.1f, 0.15f});

// At the start of each frame, use clearColorRgba() to clear the framebuffer
auto [r, g, b, a] = env.clearColorRgba();
```

`Environment` is a plain data struct. Use `clearColorRgba()` when clearing the framebuffer and pass `exposure` / `gamma` to your shaders.
