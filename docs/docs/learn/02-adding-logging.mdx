# 02: Adding Logging

The **vnetemplate** project (from [01: Hello VertexNova](../01-hello-vertexnova)) already includes **vnelogging** at `deps/internal/vnelogging` and links your app to `vne::logging`. Nothing to add — just include the header and call the API.

Full reference: **[vnelogging repository](https://github.com/vertexnova/vnelogging)**.

## Quick Start

```cpp
#include <vertexnova/logging/logging.h>

namespace {
CREATE_VNE_LOGGER_CATEGORY("app")
}  // namespace

int main() {
    vne::log::LoggerConfig config;
    config.name       = vne::log::kDefaultLoggerName;
    config.sink       = vne::log::LogSinkType::eBoth;
    config.file_path  = "app.log";
    config.log_level  = vne::log::LogLevel::eDebug;

    vne::log::Logging::configureLogger(config);

    VNE_LOG_INFO  << "Application started";
    VNE_LOG_DEBUG << "Processing items";
    VNE_LOG_ERROR << "Failed to load file";

    vne::log::Logging::shutdown();
    return 0;
}
```

| Piece | What it does |
|-------|-------------|
| `CREATE_VNE_LOGGER_CATEGORY("app")` | Sets the category label for this translation unit (appears in file logs). Put it in an anonymous namespace in each `.cpp`. |
| `LoggerConfig` | Bundles name, sink, file path, log level, and other options into one struct. |
| `Logging::configureLogger(config)` | Creates the logger and applies the config. Call once at startup before any log macros. |
| `VNE_LOG_INFO << "msg" << value` | Stream-style macro. One per log line; no `std::endl` needed. |
| `Logging::shutdown()` | Flushes and closes all sinks. Call before process exit. |

## LoggerConfig fields

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `name` | `std::string` | — | Use `vne::log::kDefaultLoggerName` (`"vertexnova"`) for the default logger. |
| `sink` | `LogSinkType` | `eConsole` | `eConsole`, `eFile`, or `eBoth`. |
| `file_path` | `std::string` | `""` | Required when sink includes `eFile`. |
| `log_level` | `LogLevel` | `eInfo` | Messages below this level are dropped. |
| `flush_level` | `LogLevel` | `eError` | Messages at or above this level flush immediately. Defaults to `eError` — errors always flush, debug output may be buffered. |
| `console_pattern` | `std::string` | `""` | spdlog pattern string for console output. Empty = library default (timestamp, level, category, message). |
| `file_pattern` | `std::string` | `""` | spdlog pattern string for file output. |
| `async` | `bool` | `false` | `true` = non-blocking background thread. Useful for high-frequency logging. |

## Log levels

| Level | Macro | When to use |
|-------|-------|-------------|
| Trace | `VNE_LOG_TRACE` | Function entry/exit, very verbose |
| Debug | `VNE_LOG_DEBUG` | Development diagnostics |
| Info  | `VNE_LOG_INFO`  | Normal operation |
| Warn  | `VNE_LOG_WARN`  | Unexpected but recoverable |
| Error | `VNE_LOG_ERROR` | Operation failed |
| Fatal | `VNE_LOG_FATAL` | Cannot continue |

Set `config.log_level = vne::log::LogLevel::eInfo` in production to suppress Debug and Trace output.

## Named loggers and categories

The default macros (`VNE_LOG_INFO`, etc.) write to `kDefaultLoggerName`. For separate loggers per subsystem, configure each one at startup and then use the `_L` or `_LC` variants:

```cpp
// Configure a second logger at startup
vne::log::LoggerConfig render_cfg;
render_cfg.name      = "render";
render_cfg.sink      = vne::log::LogSinkType::eFile;
render_cfg.file_path = "render.log";
render_cfg.log_level = vne::log::LogLevel::eTrace;
render_cfg.async     = true;
vne::log::Logging::configureLogger(render_cfg);

// Write to the named logger (category comes from CREATE_VNE_LOGGER_CATEGORY)
VNE_LOG_INFO_L("render") << "Frame " << frame_idx;

// Or specify both logger and category inline
VNE_LOG_DEBUG_LC("render", "mesh") << "Uploading " << vertex_count << " vertices";
```

| Macro family | When to use |
|---|---|
| `VNE_LOG_*` | Single logger for the whole app |
| `VNE_LOG_*_L(logger)` | Named logger; category from `CREATE_VNE_LOGGER_CATEGORY` in this file |
| `VNE_LOG_*_LC(logger, category)` | Named logger with an explicit category override |

## RAII shutdown with LoggingGuard

If you're using the testbed framework, `vne::testbed::LoggingGuard` handles configure and shutdown for you:

```cpp
#include <vertexnova/testbed/logging_guard.h>

int main() {
    vne::testbed::LoggingGuard logging_guard;
    VNE_LOG_INFO << "Application started";
    return 0;
}  // shutdown() called automatically when logging_guard goes out of scope
```

In your own projects, configure in a constructor and call `Logging::shutdown()` in the destructor for the same effect, or configure/shutdown explicitly in `main()`.

## Run the vnelogging examples (optional)

The library ships five examples you can run from the submodule:

```bash
cd deps/internal/vnelogging
cmake -B build -DBUILD_EXAMPLES=ON
cmake --build build
./build/bin/01_HelloLogging
```

| Example | What it covers |
|---------|---------------|
| `01_HelloLogging` | Basic setup — all log levels, console + file |
| `02_SubsystemLogging` | Separate loggers for app, physics, render |
| `03_SpdlogIntegration` | vnelogging and spdlog in the same process |
| `04_Benchmark` | Sync vs async throughput and latency |
| `05_Multithreaded` | Four threads logging concurrently |

Full details: [vnelogging README](https://github.com/vertexnova/vnelogging) · [Wiki](https://github.com/vertexnova/vnelogging/wiki).

## If you're not using vnetemplate

Add vnelogging as a git submodule or via CMake FetchContent, then link `vne::logging` to your target. The API above is identical. See [vnelogging README — installation](https://github.com/vertexnova/vnelogging#installation) for the CMake steps.
