# 02: Adding Logging

In [01: Hello VertexNova](./01-hello-vertexnova) you built and ran the **vnetemplate** project. That template already includes **vnelogging** in `deps/internal/vnelogging`. Here we use it: where it lives, how to call it, and how to run the hello-logging example.

No need to add vnelogging as a new dependency — your template already builds with it.

## Where Logging Lives in the Template

In your vnetemplate checkout:

| Path | Purpose |
|------|---------|
| `deps/internal/vnelogging/` | The vnelogging library (submodule) |
| Include in code | `#include <vertexnova/logging/logging.h>` |

The template's CMake already links your app or examples to `vne::logging`. You only need to configure and use the API.

## Quick Start (Same as vnelogging README)

The [vnelogging Quick Start](https://github.com/vertexnova/vnelogging#quick-start) looks like this. In your template project's `main.cpp` or an example:

```cpp
#include <vertexnova/logging/logging.h>

namespace {
CREATE_VNE_LOGGER_CATEGORY("app")
}  // namespace

int main() {
    // Configure logger
    vne::log::LoggerConfig config;
    config.name = vne::log::kDefaultLoggerName;
    config.sink = vne::log::LogSinkType::eBoth;
    config.file_path = "app.log";
    config.log_level = vne::log::LogLevel::eDebug;

    vne::log::Logging::configureLogger(config);

    // Log messages
    VNE_LOG_INFO << "Application started";
    VNE_LOG_DEBUG << "Processing items";
    VNE_LOG_ERROR << "Failed to load file";

    vne::log::Logging::shutdown();
    return 0;
}
```

| Piece | Meaning |
|--------|--------|
| `CREATE_VNE_LOGGER_CATEGORY("app")` | Category label for this file (shows up in file logs) |
| `LoggerConfig` | name, sink (console/file/both), file_path, log_level |
| `Logging::configureLogger(config)` | Apply config before logging |
| `VNE_LOG_INFO` / `VNE_LOG_DEBUG` / `VNE_LOG_ERROR` | Stream-style macros: `VNE_LOG_INFO << "msg" << value;` |
| `Logging::shutdown()` | Flush and close; call before exit |

## Use Logging in Your Template Code

In the template's main app or an example target that already links `vne::logging`:

1. **Include** the header and **create a category** (in one .cpp or per file):

```cpp
#include <vertexnova/logging/logging.h>

namespace {
CREATE_VNE_LOGGER_CATEGORY("hello")
}  // namespace
```

2. **Configure once** (e.g. at startup):

```cpp
vne::log::LoggerConfig config;
config.name = vne::log::kDefaultLoggerName;
config.sink = vne::log::LogSinkType::eBoth;  // Console + file
config.file_path = "logs/hello.log";
config.log_level = vne::log::LogLevel::eDebug;
vne::log::Logging::configureLogger(config);
```

3. **Log with the stream API**:

```cpp
VNE_LOG_INFO << "Welcome to VertexNova";
VNE_LOG_DEBUG << "Arguments: " << argc;
VNE_LOG_WARN << "No config file found, using defaults";
VNE_LOG_ERROR << "Failed to open " << path;
```

4. **Shutdown before exit**:

```cpp
vne::log::Logging::shutdown();
```

Or you can use a **LoggingGuard** to configure logging in its constructor and call `Logging::shutdown()` in its destructor, so you don't have to do it in main.

### Alternatively: LoggingGuard

Define a guard that configures on construction and shuts down on destruction:

```cpp
class LoggingGuard {
 public:
  LoggingGuard() {
    vne::log::LoggerConfig config;
    config.name = vne::log::kDefaultLoggerName;
    config.sink = vne::log::LogSinkType::eConsole;
    config.console_pattern = "[%l] [%n] %v";
    config.log_level = vne::log::LogLevel::eInfo;
    config.async = false;

    vne::log::Logging::configureLogger(config);
  }

  ~LoggingGuard() { vne::log::Logging::shutdown(); }

  LoggingGuard(const LoggingGuard&) = delete;
  LoggingGuard& operator=(const LoggingGuard&) = delete;
};
```

Then in `main()` create a guard at scope and use logging as usual; shutdown runs automatically when the guard goes out of scope:

```cpp
int main() {
  LoggingGuard guard;
  VNE_LOG_INFO << "Application started";
  // ... rest of app; shutdown() called when guard is destroyed
  return 0;
}
```

Rebuild your template project and run — you'll get timestamps, colors in the console, and the same messages in the log file.

## Log Levels

| Level | Macro | When to use |
|-------|--------|-------------|
| Trace | `VNE_LOG_TRACE` | Very verbose (e.g. function entry/exit) |
| Debug | `VNE_LOG_DEBUG` | Development debugging |
| Info | `VNE_LOG_INFO` | Normal operation |
| Warn | `VNE_LOG_WARN` | Something unusual but not broken |
| Error | `VNE_LOG_ERROR` | Something failed |
| Fatal | `VNE_LOG_FATAL` | Cannot continue |

Set `config.log_level = vne::log::LogLevel::eInfo` in production to hide DEBUG/TRACE.

## More in vnelogging

| Feature | Description |
|---------|-------------|
| **Async logging** | `config.async = true` — non-blocking |
| **Multiple loggers** | Separate loggers for physics, render, audio |
| **Patterns** | Customize timestamp/level/category format |
| **Cross-platform** | Linux, macOS, Windows, iOS, Android, Web |

Full API and design: **[vnelogging repository](https://github.com/vertexnova/vnelogging)** (README, Wiki, examples).

## If You Built from Scratch Instead

If you followed the optional [Build from scratch](./01-hello-vertexnova#optional-build-from-scratch) path in 01 and have a minimal project **without** vnetemplate, you can add vnelogging in a few steps. Each step below is in its own box so you can follow them in order.

:::info Step 1 — Add vnelogging to your project

Choose one way to bring vnelogging into your repo.

**Option A — Git submodule (clone lives in your repo):**

From your project root:

```bash
mkdir -p external
git submodule add https://github.com/vertexnova/vnelogging.git external/vnelogging
git submodule update --init --recursive
```

Then in your **`CMakeLists.txt`**, before `add_executable`, add:

```cmake
add_subdirectory(external/vnelogging)
```

**Option B — FetchContent (CMake downloads at configure time):**

See the [vnelogging README — FetchContent](https://github.com/vertexnova/vnelogging#option-2-fetchcontent) section. You add a `FetchContent_Declare` / `FetchContent_MakeAvailable` block and then `add_subdirectory` on the populated source dir. No git submodule required.

:::

:::info Step 2 — Link your executable to vnelogging

After `add_executable(YourTargetName src/main.cpp)` (or whatever your target is), link the logging library:

```cmake
add_executable(HelloVertexNova src/main.cpp)
target_link_libraries(HelloVertexNova PRIVATE vne::logging)
```

Use your actual target name in place of `HelloVertexNova`. Re-run CMake and build; your target will have the vnelogging headers and library.

:::

:::info Step 3 — Use the logging API in your code

From here on, use the same API as in the rest of this chapter:

1. **Include** the header and **create a category** (e.g. in `main.cpp`):  
   `#include <vertexnova/logging/logging.h>` and `CREATE_VNE_LOGGER_CATEGORY("app")` in an anonymous namespace.
2. **Configure** once at startup: set up `vne::log::LoggerConfig` (name, sink, log_level, etc.) and call `vne::log::Logging::configureLogger(config)`.
3. **Log** with the stream macros: `VNE_LOG_INFO << "message";`, `VNE_LOG_DEBUG`, `VNE_LOG_ERROR`, etc.
4. **Shutdown** before exit: `vne::log::Logging::shutdown();` — or use a [LoggingGuard](#alternatively-loggingguard) so shutdown runs automatically.

Rebuild and run; you should see the same colored, timestamped output (and optional file logging) as in the template.

:::

For full details, options, and advanced usage, see the [vnelogging README](https://github.com/vertexnova/vnelogging).
