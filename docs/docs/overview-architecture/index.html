Architecture

VertexNova is organized as a modular C++17/20 stack — each layer is independently buildable, testable, and documented. Dependencies only flow downward (L4 → L3 → L2 → L1).

Layer Summary​

LayerResponsibility
L4 ApplicationEditor, demos, tests, viewer — programs that use the engine
L3 Graphicsvnecrossgl, vnecrosswin, vnecrossviz, vnecrossshader — multi-backend rendering, windowing, shaders, visualisation (private)
L2 CoreMath, logging, events, I/O, scene, input — platform-independent utilities
L1 PlatformVulkan, Metal, OpenGL, SPIRV-Cross, GLM, GLFW — external deps and targets
Public vs private

The Graphics layer (vnecrossgl, vnecrosswin, vnecrossviz, vnecrossshader) is developed as private libraries. The public vnetestbed repository provides demos and samples that use this stack.

Dependency Rule​

  • Core (L2) depends only on Platform (L1) and each other
  • Graphics (L3) depends on Core and graphics APIs
  • Application (L4) depends on Graphics and Core
  • Nothing in a lower layer may include a header from a higher layer

This keeps every library independently buildable. Lower layers do not know that higher layers exist.
