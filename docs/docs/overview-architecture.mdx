# Architecture

VertexNova is organized as a **modular C++20 stack** — each layer is independently buildable, testable, and documented. Dependencies only flow downward (L4 → L3 → L2 → L1).

## Layer Summary

| Layer | Responsibility |
|-------|-----------------|
| **L4 Application** | Editor, demos, tests, viewer — programs that use the engine |
| **L3 Graphics** | Rendering, windowing, shaders, visualisation — multi-backend abstraction |
| **L2 Core** | Math, logging, events, I/O, scene, input — platform-independent utilities |
| **L1 Platform** | Vulkan, Metal, OpenGL, SPIRV-Cross, GLM, GLFW — external deps and targets |

## Dependency Rule

- **Core (L2)** depends only on Platform (L1) and each other
- **Graphics (L3)** depends on Core and graphics APIs
- **Application (L4)** depends on Graphics and Core
- **Nothing in a lower layer may include a header from a higher layer**

This keeps every library independently buildable. Lower layers do not know that higher layers exist.
